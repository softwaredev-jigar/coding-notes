What is React?

React (or ReactJS) is an open-source JavaScript library developed by Facebook for building interactive user interfaces efficiently using a component-based architecture. It lets developers create reusable UI components, making development fast, scalable, and organized.

Key Features of React

  • JSX (JavaScript XML): JSX is a syntax extension for JavaScript that allows writing HTML-like code in JavaScript files, making the code readable and maintainable.
  • Component-Based Architecture: React breaks the UI into reusable, independent components, simplifying code organization and reuse.
  • Virtual DOM: React uses a virtual DOM to update only the changed parts of the real DOM efficiently, improving performance.
  • One-Way Data Binding: Data in React flows from parent to child components via props, ensuring predictable and easy-to-debug UI updates.
  • State Management: State is used to handle dynamic data and is automatically tracked and re-rendered by React components.
  • Hooks: Hooks (like useState, useEffect) enable functional components to use state and lifecycle features, replacing class component methods in modern React.
  • High Performance: Only affected components are updated, making React applications fast and efficient.

Why React?

React is suitable for developing scalable applications. It encourages a clean separation of concerns by splitting UI into logical components and has a large ecosystem with helpful developer tools, libraries, and a supportive community.

Quick code example (functional component)

            
                import React, { useState } from 'react';

                function Counter() {
                const [count, setCount] = useState(0);

                return (
                    <div>
                    <h2>Count: {count}</h2>
                    <button onClick={() => setCount(count + 1)}>
                        Increment
                    </button>
                    </div>
                );
                }
                export default Counter;

‚öôÔ∏è Real DOM vs Virtual DOM

React uses a Virtual DOM ‚Äî a lightweight copy of the real DOM ‚Äî to efficiently update only the parts of the UI that change.

FeatureReal DOMVirtual DOM
Update SpeedSlower (entire UI re-rendered)Faster (only changed parts)
PerformanceHeavy DOM operationsOptimized updates
MemoryHigher usageLower usage
React‚Äôs RoleUses directlyManages updates efficiently

How it works:

  1. React keeps a copy of the real DOM in memory (Virtual DOM).
  2. When something changes, React compares old vs new Virtual DOM.
  3. Only the changed elements are updated in the Real DOM.

‚öõÔ∏è What is a Component?

A Component in React is an independent, reusable piece of UI. Think of it like a small function that returns HTML. Each component manages its own structure, style, and logic ‚Äî making your app modular and easy to maintain.

  • There are two types: Functional Components and Class Components.
  • Functional components are most commonly used today (with Hooks).

function Welcome() {
  return (
    <h2>Hello, User!</h2>
  );
}

export default Welcome;


import Welcome from './Welcome';

function App() {
  return (
    <div>
      <Welcome />
    </div>
  );
}

üéØ What are Props?

Props (Properties) are used to pass data from one component to another ‚Äî usually from a parent component to a child component. They make components dynamic and reusable.

  • Props are read-only ‚Äî you cannot modify them inside the child component.
  • They work just like function parameters.

function Welcome({ name }) {
  return (
    <h2>Hello, {name}!</h2>
  );
}

export default Welcome;


import Welcome from './Welcome';

function App() {
  return (
    <div>
      <Welcome name="Jigar" />
      <Welcome name="React Learner" />
    </div>
  );
}

üí® Tailwind CSS Installation

Tailwind CSS is a utility-first CSS framework for rapidly building modern UIs. It works perfectly with React projects created using Vite or Create React App.

‚öôÔ∏è Steps to Install Tailwind (for Vite or CRA)


# 1. Install Tailwind via npm
npm install -D tailwindcss postcss autoprefixer

# 2. Initialize Tailwind config files
npx tailwindcss init -p


# 3. Configure template paths (tailwind.config.js)
content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"],


# 4. Add Tailwind to your CSS (index.css)
@tailwind base;
@tailwind components;
@tailwind utilities;


# 5. Start your project
npm run dev
